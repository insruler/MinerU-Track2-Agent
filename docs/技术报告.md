## 一、系统概述

**惯导智衡**是面向惯性产品检测领域的智能 Data Agent 系统，基于 MinerU 云端 API 构建，实现从原始 PDF 检测报告到结构化知识图谱、智能问答、多步任务自动执行的完整闭环。

### 核心能力

| 能力维度 | 描述 |
|---------|------|
| 文档解析 | MinerU 云端 API 解析 PDF，提取 Markdown 结构化内容 |
| 知识抽取 | MiniMax LLM 驱动的实体/关系抽取，构建领域知识图谱 |
| 智能问答 | RAG 检索增强（TF-IDF+BM25），基于知识库精准回答 |
| Agent 执行 | ReAct 模式（Reason-Act-Observe），支持复杂多步骤任务 |
| 可视化 | 知识图谱 D3 力导向可视化，支持拖拽/缩放/节点检索 |

## 三、核心技术实现

### 3.1 ReAct Agent（赛道核心）

采用 **Reason-Act-Observe** 循环架构，每步包含：
1. **Reason**：基于当前上下文进行 Chain-of-Thought 推理
2. **Act**：选择并调用合适工具（6个注册工具）
3. **Observe**：观察工具返回结果，决定下一步

```python
# ReAct 核心循环（最多5步）
for step in range(max_steps):
    thought = llm_reason(context, history)      # 推理
    action, action_input = parse_action(thought) # 解析动作
    observation = execute_tool(action, action_input) # 执行
    history.append((thought, action, observation))   # 记录
    if is_final_answer(thought):
        break
```

**已注册工具**：
- `search_knowledge_base`：RAG分块检索
- `get_kg_entities`：知识图谱实体查询
- `get_kg_relations`：实体关系查询
- `extract_structured_fields`：结构化字段抽取
- `call_llm`：LLM推理调用
- `get_stats`：系统统计查询

### 3.2 RAG 检索增强问答

**分块策略**：
- 块大小：512 字符
- 重叠：128 字符
- 最小过滤：50 字符
- 总分块数：28,741 块（覆盖 356 篇文档）

**检索流程**：
```
用户问题 → TF-IDF 向量化 → 候选块召回 → BM25 重排序 → Top-K 块
         → 构建上下文 → LLM 推理 → 带引用溯源的回答
```

**引用溯源**：回答中自动标注文档来源 `[1][2]`，可追溯到具体检测报告。

### 3.3 惯性检测领域知识图谱

**实体类型**：
- 惯性器件：陀螺仪、加速度计、IMU、惯性导航系统
- 测试参数：标度因数、零偏稳定性、随机游走、Allan方差
- 测试方法：角振动试验、温度循环、冲击振动
- 标准规范：GJB系列、QJ系列、YDT系列

**构建流程**：MinerU解析 → LLM实体抽取 → 关系识别 → SQLite存储 → D3可视化

### 3.4 MinerU 云端高精度解析

- **API端点**：`https://mineru.net/api/v4`
- **支持格式**：PDF / DOCX / PPTX / XLSX
- **解析能力**：表格识别、公式提取、图表描述
- **已处理**：356篇惯性检测报告（共300+ PDF）

## 四、系统稳定性设计

### 4.1 守护进程
```ini
# /etc/systemd/system/insruler.service
[Service]
Restart=always
RestartSec=5
```
服务崩溃后5秒自动重启，已连续稳定运行4天+。

### 4.2 全链路追踪
每个请求自动生成 `Correlation-ID`，贯穿整个处理链路：
```
Request → Middleware → Handler → Tool → LLM → Response
   └─────────── correlation_id: f3bda0fb ──────────────┘
```

### 4.3 参数校验
所有 API 入参使用 Pydantic v2 模型校验，支持字段别名兼容：
- `query` / `question` 均可用于问答接口
- `query` / `task` 均可用于 Agent 接口

### 4.4 异步执行
Agent 执行采用异步模式，消除 HTTP 超时风险：
```
POST /api/agent/execute → 立即返回 trace_id
GET  /api/agent/result?trace_id=xxx → 轮询结果
```
