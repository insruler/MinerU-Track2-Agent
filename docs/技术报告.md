## 一、系统概述

**惯导智衡**是面向惯性产品检测领域的智能 Data Agent 系统，基于 MinerU 云端 API 构建，实现从原始 PDF 检测报告到结构化知识图谱、智能问答、多步任务自动执行的完整闭环。

### 核心能力

| 能力维度 | 描述 |
|---------|------|
| 文档解析 | MinerU 云端 API 解析 PDF，提取 Markdown 结构化内容 |
| 知识抽取 | MiniMax LLM 驱动的实体/关系抽取，构建领域知识图谱 |
| 智能问答 | RAG 检索增强，基于知识库精准回答惯性检测领域问题 |
| Agent 执行 | 两阶段任务规划+执行，支持复杂多步骤自动化任务 |
| 可视化 | 知识图谱 D3 力导向可视化，支持拖拽/缩放/节点检索 |

### 当前知识库规模（2026-05-21）

- 知识文档：**56 篇**（惯性检测报告）
- 实体节点：**701 个**（参数、器件、测试方法、标准等）
- 关系连接：**704 条**（用于、具有指标、规定、影响等）
- 解析任务：**150 个**（13 完成，持续扩充中）

## 三、核心模块详解

### 3.1 PDF 解析模块

采用 MinerU 云端 API v4 的 VLM 模型进行文档解析：

```
上传 PDF → 获取预签名 OSS URL → PUT 上传 → 提交解析任务
→ 轮询状态（5s间隔，最长3分钟）→ 下载 ZIP → 提取 Markdown
```

**关键实现**（`mineru_parse_pdf` 函数）：
1. 调用 `/api/v4/file-urls/batch` 获取上传 URL 和 batch_id
2. PUT 上传 PDF 到 OSS
3. 轮询 `/api/v4/extract-results/batch/{batch_id}` 等待完成
4. 下载 ZIP 包，提取 `.md` 文件作为解析结果

**支持格式**：PDF（含扫描件、表格、公式）

### 3.2 知识图谱构建模块

解析完成后，自动调用 MiniMax LLM 进行实体关系抽取：

**Prompt 设计**：
```
你是一个惯性技术领域的知识图谱抽取专家。
实体类型：惯性器件、技术参数、测试方法、设备型号、人名、机构名等
关系类型：用于、具有指标、规定、影响、包含、使用、导致、属于等
输出严格 JSON 格式：{"entities": [...], "relations": [...]}
```

**去重策略**：实体名称唯一性约束，关系三元组（源、目标、类型）唯一性约束

**当前图谱统计**：
- 实体类型分布：参数(94)、测试方法(48)、误差(43)、国军标(36)、人名(33)、器件(27)...
- 关系类型分布：用于(74)、具有指标(62)、规定(41)、影响(34)、包含(32)...

### 3.3 Data Agent 模块

采用**两阶段规划-执行架构**：

**第一阶段：任务规划**
- 接收用户自然语言任务描述
- 检索知识库获取相关文档上下文
- LLM 生成结构化执行计划（任务理解→步骤规划→预期结果）

**第二阶段：计划执行**
- 将第一阶段生成的计划作为上下文传入执行接口
- LLM 严格按计划执行，输出分析结论、数据摘要、改进建议
- 支持 Markdown 格式输出，便于结构化展示

**工具调用能力**：
- `search_docs(query)` — 知识库语义检索
- `get_kg_entities()` — 知识图谱实体查询
- `mineru_parse_pdf()` — 文档解析工具


### 3.5 工具链注册系统（v4.0新增）

采用注册式工具链架构，Agent通过工具调用完成复杂任务：

| 工具名称 | 功能描述 |
|---------|---------|
| `search_knowledge_base` | 多关键词分词检索，提升召回率 |
| `get_kg_entities` | 按类型/名称查询知识图谱实体 |
| `get_kg_relations` | 查询实体关联关系 |
| `extract_structured_fields` | LLM提取报告编号/检测项目/合格判定 |
| `call_llm` | 调用MiniMax LLM进行推理分析 |
| `get_stats` | 获取系统知识库统计信息 |

每次工具调用均记录到 `agent_execution_logs` 表，支持通过 `trace_id` 全链路追踪。

### 3.6 全文分段实体抽取（v4.0升级）

原版本只取文档前3000字进行实体抽取，v4.0升级为分段全文抽取：

- 分块大小：3000字/块，200字重叠（保留上下文）
- 最多处理：8块（约24,000字）
- 去重策略：实体名称+类型唯一性约束，关系三元组唯一性约束
- 效果：实体覆盖率从约30%提升至接近100%

### 3.4 RAG 智能问答模块

```
用户问题 → 关键词检索知识库（title+content LIKE匹配）
→ 取 Top-3 相关文档片段（各500字）
→ 构建 Prompt（问题 + 知识库上下文）
→ MiniMax LLM 生成答案
→ 持久化到 qa_history 表
→ 返回答案 + 来源文档列表
```

## 四、典型任务执行示例（≥5个）

### 示例 1：陀螺仪零偏稳定性检测标准查询

**输入**：陀螺仪零偏稳定性的检测标准是什么？

**执行过程**：
1. 检索知识库，匹配相关文档
2. 构建包含领域知识的 Prompt
3. LLM 生成专业回答

**输出摘要**：
> 陀螺仪零偏稳定性检测主要采用 Allan 方差分析法，通过对不同相关时间下的方差分析识别角度随机游走、零偏稳定性等5种噪声项。高精度陀螺仪零偏稳定性可达 0.001 deg/h 级别，依据 GJB 150A-2009 标准执行环境试验。

## 五、系统性能与稳定性

### 性能指标

| 指标 | 数值 |
|------|------|
| API 响应时间（stats/kg/docs） | < 100ms |
| PDF 解析时间（MinerU云端） | 30~180s（依文档复杂度） |
| LLM 实体抽取时间 | 10~30s |
| QA 问答响应时间 | 5~15s |
| 并发支持 | Uvicorn 多进程，支持多用户同时访问 |

### 稳定性设计

1. **任务异步化**：所有耗时操作（PDF解析、LLM调用）均在后台线程执行，不阻塞 API 响应
2. **状态持久化**：任务状态、解析结果、QA历史全部持久化到 SQLite，服务重启不丢失
3. **错误恢复**：MinerU 解析失败自动记录错误信息，支持重试；LLM 调用异常不影响主流程
4. **轮询优化**：前端轮询在所有任务完成（done+failed=total）后自动停止，避免无效请求
5. **内存安全**：文档内容截断至 200,000 字符存储，防止超大文档导致内存溢出

### 工程可复现性

- 完整 `requirements.txt` 依赖声明
- 单文件启动（`python3 app.py`）
- SQLite 零配置数据库
- 环境变量配置（`MINIMAX_API_KEY`、`MINERU_TOKEN`）

## 六、应用场景与价值

### 核心应用场景

1. **检测报告智能归档**：批量解析 PDF 检测报告，自动提取关键信息入库，替代人工录入
2. **技术标准知识库**：将 GJB、ISO 等标准文档解析入库，支持快速标准查询
3. **跨报告参数对比**：自动对比多份检测报告的技术参数，辅助质量分析
4. **检测流程辅助**：基于历史报告和标准知识，为检测工程师提供实时技术支持
5. **知识图谱构建**：自动构建惯性检测领域知识图谱，支持关系推理和知识发现

### 产业价值

- **效率提升**：将人工阅读报告的时间从小时级压缩到分钟级
- **知识沉淀**：将分散在 PDF 中的专业知识结构化，形成可复用的领域知识库
- **泛化能力**：架构通用，可快速迁移到其他检测领域（电子、材料、机械等）

## 七、系统局限性与改进方向

1. **检索精度**：当前使用关键词 LIKE 匹配，后续可引入向量嵌入实现语义检索
2. **实体抽取准确率**：依赖 LLM 输出，存在幻觉风险，后续可引入规则校验层
3. **并发解析**：当前串行处理批量任务，后续可引入任务队列（Celery/Redis）提升吞吐
4. **多模态支持**：当前仅支持 PDF，后续可扩展 DOCX、XLSX、图片等格式
